# Task 19 Report: Resource Library Web App

## Overview

This project involves building a simple web application using Express.js on Node.js.
The application allows users to:
- Register and log in
- View available resources (articles/books)
- Add, edit, and delete resources

## Technologies Used
- Node.js – runtime environment (place where code actually runs)
- Express.js – web framework (basic structure to build websites easier and faster)
- EJS – templating engine for UI
- Express-session – user authentication (lets app remember users)
- Body-parser – handling form data (used to read data from forms)

## Features Implemented
- User Management (register, login, logout)
- Resource Management (CRUD - create, read, update, delete)

## Key Concepts

### 1. What is Express.js?

Express.js is a lightweight web framework for Node.js used to build web applications and APIs.

In this project, Express is used to:

Create routes (/login, /add, etc.)
Handle form submissions
Render web pages using EJS

### 2. Difference Between Node.js and JavaScript
JavaScript	|   Node.js
|Programming language |  Runtime environment |
|Runs in browser	Runs |   outside browser     |
|Used for frontend	 |  Used for backend     |

JavaScript = the language
Node.js = tool that lets JavaScript run on your computer/server

### 3. Website vs Webpage vs Web App
Term	Meaning
Webpage	A single page (like one screen)
Website	Collection of webpages
Web App	Interactive website with functionality

Think of a restaurant:

Webpage → One menu page
Website → The full menu book
Web App → A self-ordering system where you log in and place orders
